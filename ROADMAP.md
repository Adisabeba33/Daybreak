## Совместная работа над планом (Share, вариант B)

**Суть:** пользователь делится планом с помощником, и тот не просто читает
снимок, а **работает над планом** — отмечает задачи, дописывает, и всё
синхронизируется обратно владельцу.

**Что для этого нужно (это скачок «из соло в мульти»):**
- Аккаунты / вход (пока приложение локальное, на `localStorage`).
- Бэкенд + хранилище планов на сервере (Supabase/Firebase — обсуждали ранее).
- Синхронизация в реальном времени между владельцем и помощником.
- Хостинг плана по ссылке — красивая веб-страница `daybreak.app/p/<id>`,
  которую видит получатель (и read-only режим для тех, у кого нет аккаунта).
- Виральный крючок: ссылка «подпишись, чтобы планировать так же» реально
  заработает только когда появятся аккаунты + хостинг плана.

**Отношение к варианту A:** «снимок» плана через Web Share (текст + ссылка,
read-only) — дёшево и самодостаточно; вариант B надстраивается над ним.

**Статус:** отложено. Делать, когда решим переходить на аккаунты/бэкенд.
